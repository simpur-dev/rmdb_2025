# RMDB 开发状态

## 项目概述
RMDB（Renmin Database）是一个关系型数据库管理系统，为全国大学生计算机系统能力大赛数据库管理系统赛道提供代码框架。

## 当前开发进度

### ✅ 已完成模块

#### 1. 存储层（Storage Layer）
- **disk_manager.cpp** - 磁盘管理器
  - 实现 `write_page()` - 将数据写入磁盘指定页面
  - 实现 `read_page()` - 从磁盘读取指定页面数据
  - 实现 `allocate_page()` - 分配新的页号
  - 实现 `create_file()` - 创建数据库文件
  - 实现目录和文件管理功能

- **buffer_pool_manager.cpp** - 缓冲池管理器
  - 实现 `fetch_page()` - 从缓冲池获取页面
  - 实现 `find_victim_page()` - 使用 LRU 策略寻找可淘汰页面
  - 实现 `update_page()` - 更新页面数据（脏页写回）
  - 实现 `unpin_page()` - 取消固定页面
  - 实现 `flush_page()` - 将页面写回磁盘
  - 实现 `new_page()` - 创建新页面
  - 实现 `delete_page()` - 删除页面

#### 2. 页面替换策略（Replacer）
- **lru_replacer.cpp** - LRU 页面替换器
  - 实现 `victim()` - LRU 策略淘汰最近最少使用页面
  - 实现 `pin()` - 固定页面，防止被淘汰
  - 实现 `unpin()` - 取消固定页面，允许被淘汰
  - 实现 `Size()` - 获取可淘汰页面数量

## 技术实现要点

### LRU 策略实现
- 使用双向链表（`LRUlist_`）和哈希表（`LRUhash_`）实现 O(1) 时间复杂度的 LRU
- 链表头部表示最近访问，尾部表示最久未使用
- `pin()` 操作从数据结构中移除页面
- `unpin()` 操作将页面加入链表头部

### 缓冲池管理
- 维护空闲帧列表（`free_list_`）和 LRU 替换器
- 支持脏页自动写回机制
- 引用计数（`pin_count_`）控制页面生命周期
- 线程安全的并发控制（使用 `std::scoped_lock`）
